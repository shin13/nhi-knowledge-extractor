# Changelog

All notable changes to **the `nhi-knowledge-extractor` pipeline itself** are
documented here. For per-release NHI data diffs (added / modified / removed
regulation items), see `data/regulations/medication/CHANGELOG_data.md`, which
is regenerated by every `nhi-extract sync` run and not tracked in git.

Format based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
This project follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- `EMIT_DEPTH` parameter (default 5) — decouples editorial chunk granularity
  from token budget. Chunker now forces descent past nodes shallower than this
  depth even if the subtree fits, preventing "整節一筆" collapse when budgets
  scale. CLI flag `--emit-depth N` available on `sync` and `chunk`.
- Item metadata fields `parent_id` / `part_index` / `total_parts` and the
  corresponding CSV columns. Lets downstream RAG hydrate split siblings when
  one part is retrieved.
- Anchor preamble in Strategy 0 recursive sub-split: continuation sub-parts
  (e.g. `sec9-9.69-part3-2`) now inject the group opener line followed by
  `（續）：` so each row is self-contained for retrieval.
- GitHub Actions CI running pytest on every push and PR.
- `docs/roadmap.md` — categorised future plan (P0–P6).
- `docs/emit-depth-plan.md` — design rationale for the chunker overhaul.

### Changed
- CSV schema: 8 → 11 columns. New columns: `parent_id`, `part_index`,
  `total_parts`. Existing columns unchanged. **Breaking** for any consumer
  that reads CSVs by positional index; `csv.DictReader` and pandas readers
  are transparent.
- `CHANGELOG.md` renamed responsibility: now records *project* version
  history, not pipeline-generated data-release diffs. The data log moved to
  `data/regulations/medication/CHANGELOG_data.md` (gitignored, pipeline-owned).
- README rewritten with English + Traditional Chinese versions, quickstart
  with example output, and a worked CSV row example.

### Fixed
- 9 / 512 rows that previously merged multiple drug regulations into one
  chunk (1.8% pollution) now correctly split into per-drug rows. Worst case
  fixed: `sec5-5.1` 糖尿病用藥 (was 12 sub-headings in one row, now 12 rows).

## Pre-history

This project's pre-public-release work is captured in git commits before tag
`v0.1.0`. Highlights:

- DOCX/ODT parser with native table extraction (replaces predecessor's
  Google Docs roundtrip for embedded tables).
- Variable-depth chunker with hard token budget contract.
- Multi-format NHI fetcher (DOCX + ODT, 17 regulation chapters).
- Strategy 0 leaf-splitter for over-budget numbered-item bodies.
- Same-date CHANGELOG replace semantics in packager.
- Tilde cross-reference rejection in heading detection.
